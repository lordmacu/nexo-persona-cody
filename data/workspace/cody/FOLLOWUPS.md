# Follow-ups

This file tracks the **active technical backlog** for your project.

Cody reads individual entries via `followup_detail code=<X>` and
can dispatch them as standalone goals via `dispatch_followup
code=<X>`.

Entry shape:

```
### <code> — <one-line title>

<one paragraph describing the deuda or follow-up. Add Why + How
to apply lines if helpful.>
```

The starter is empty — add entries as your project's audit /
refactor / cleanup work surfaces.
